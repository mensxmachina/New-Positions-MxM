3.	Expression of Interest for AI Engineer  Position (EIC Transition Project)
Role Overview
We are building an agentic data analytics platform in Python, where autonomous agents orchestrate data processing, model execution, and decision-making workflows.
We are looking for an AI Architect to design and implement the agent layer—the system that coordinates tools, ML models, and reasoning workflows into a coherent, reliable product.
This is not a pure research role. It is a systems architecture + applied AI engineering role focused on building robust, production-grade agent systems.
Key Responsibilities
•	Design and implement the agent architecture (planning, reasoning, tool use, memory) 
•	Define how the agent interacts with ML models, backend APIs and infrastructure, and external tools and data sources 
•	Build tooling frameworks for agent capabilities (function calling, tool routing, execution control) 
•	Architect state and memory systems (short-term context, long-term storage, retrieval strategies) 
•	Develop prompting and reasoning strategies (structured prompts, chain-of-thought abstractions, planning loops) 
•	Ensure robustness and reliability (guardrails, fallbacks, error handling, retries) 
•	Define evaluation frameworks for agent performance and correctness 
•	Collaborate closely with the rest of the RnD team (algorithm development, ML integration, and infrastructure) 
Required Skills
•	Strong Python engineering skills 
•	Hands-on experience building LLM-powered systems or agents 
•	Deep understanding of, tool use / function calling, Retrieval-Augmented Generation (RAG), and prompt engineering and structured outputs 
•	Experience designing multi-step reasoning workflows 
•	Familiarity with system design and distributed architectures 
•	Ability to translate ambiguous product goals into technical architectures 
Nice to Have
•	Experience with agent frameworks (LangChain, LlamaIndex, or similar) 
•	Knowledge of vector databases and retrieval systems 
•	Experience with evaluation of LLM systems (benchmarks, human-in-the-loop, automated evals) 
•	Familiarity with cost optimization strategies for LLM usage 
What Success Looks Like
•	A modular, extensible agent architecture that cleanly integrates tools, ML models, and APIs 
•	Reliable agent behavior with strong guardrails and minimal hallucination impact 
•	Efficient orchestration of workflows (low latency, controlled cost) 
•	Clear evaluation metrics and continuous improvement loops 
•	Smooth collaboration with backend and ML layers
