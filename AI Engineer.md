## Expression of Interest for AI Engineer Position (EIC Transition Project)

## Role Overview
We are building an agentic data analytics platform in Python, where autonomous agents orchestrate data processing, model execution, and decision-making workflows.

We are looking for an AI Architect to design and implement the agent layer—the system that coordinates tools, ML models, and reasoning workflows into a coherent, reliable product.

This is not a pure research role. It is a systems architecture + applied AI engineering role focused on building robust, production-grade agent systems.

## Key Responsibilities
- Design and implement agent architecture (planning, reasoning, tool use, memory)  
- Define interactions between agents, ML models, APIs, and infrastructure  
- Build tooling frameworks for agent capabilities (function calling, routing, execution control)  
- Architect state and memory systems (context, long-term storage, retrieval)  
- Develop prompting and reasoning strategies  
- Ensure robustness (guardrails, fallbacks, retries)  
- Define evaluation frameworks for agent performance  
- Collaborate with R&D team  

## Required Skills
- Strong Python engineering skills  
- Experience building LLM-powered systems or agents  
- Deep understanding of tool use, RAG, prompt engineering  
- Experience designing multi-step reasoning workflows  
- Familiarity with distributed systems  
- Ability to translate product goals into architecture  

## Nice to Have
- Experience with LangChain, LlamaIndex  
- Knowledge of vector databases  
- Experience evaluating LLM systems  
- Cost optimization for LLM usage  

## What Success Looks Like
- Modular, extensible agent architecture  
- Reliable behavior with strong guardrails  
- Efficient orchestration (low latency, controlled cost)  
- Clear evaluation metrics  
- Strong collaboration across teams
